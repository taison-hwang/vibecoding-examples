# [최종] AI 기반 스크럼 요약 및 이메일 발송 시스템 요구사항 정의서

## 1. 프로젝트 배경 및 목적

* **배경:** 매일 진행되는 스크럼 회의 내용을 기록하고, 액션 아이템을 도출하여 팀원들에게 전송하는 반복적인 수작업에서 오는 피로감을 해결하고자 합니다.
* **목적:** Gemini API를 활용해 대화체 회의록을 깔끔한 마크다운 형식으로 요약하고, 백엔드 환경변수에 저장된 공용 Gmail 계정을 통해 지정된 수신자들에게 한 번에 메일을 발송하는 단일 페이지 웹 애플리케이션을 구축합니다.
* **학습 목표:** AI API(Gemini) 연동, 웹 환경에서의 환경변수(`.env`) 보안 관리, 그리고 이메일 발송(SMTP) 라이브러리 활용 등 풀스택 미니 프로젝트의 핵심 흐름을 직접 구현하며 학습합니다.

## 2. 대상 사용자 및 시나리오

* **대상 사용자:** 스크럼 회의록 작성을 담당하는 팀원 또는 스크럼 마스터
* **이용 시나리오:** 회의 종료 후, 전사(STT)된 텍스트를 복사하여 웹앱에 붙여넣고 [AI 요약] 버튼 클릭 $\rightarrow$ 도출된 결과를 확인하고 수신자 이메일을 입력한 뒤 [이메일 전송] 버튼 클릭.

---

## 3. 구현 범위 및 단계 (MVP 우선)

### 1단계: MVP (최소 기능 제품) — 본 프로젝트의 타깃

* **회의 텍스트 입력 및 전달:** 일상어가 담긴 긴 텍스트를 입력받아 백엔드를 거쳐 Gemini API로 전달.
* **AI 요약 및 구조화:** 프롬프트 엔지니어링을 통해 결과물을 정해진 마크다운 형식으로 강제 출력.
* **단일 화면(SPA) 결과 출력:** AI가 리턴한 마크다운을 화면 우측 결과창에 즉시 렌더링.
* **이메일 주소 입력 및 검증:** 쉼표(,)로 구분된 이메일 문자열을 입력받아 프론트엔드에서 1차 검증.
* **Gmail 일괄 발송:** 시스템 대표 계정(앱 비밀번호 설정)을 사용하여 수신자들에게 텍스트 형식의 메일 발송.

### 2단계: 확장 기능 (MVP 완료 후 진행)

* **마이크 녹음 기능:** 브라우저 내장 오디오 API를 이용해 회의를 녹음하고 오디오 파일을 Gemini에 전달하여 요약.
* **로컬 저장소 활용:** 자주 발송하는 팀원의 이메일 목록을 브라우저 `LocalStorage`에 저장하여 체크박스로 선택.

## 4. 핵심 AI 프롬프트 및 데이터 규격 (Sample I/O)

개발 시 프론트엔드와 백엔드의 인터페이스 동기화를 위해 AI 출력 형식을 마크다운으로 고정합니다.

* **AI 프롬프트 규칙:** 1. 회의록을 요약하여 반드시 아래의 형식을 지켜 마크다운으로 답변할 것.
2. 특정 인물이 언급되지 않은 액션 아이템은 [공통]으로 분류할 것.
3. 액션 아이템이 전혀 없다면 *"금일 회의에서 도출된 액션 아이템이 없습니다."* 라고 출력할 것.

### 데이터 예시 (Sample I/O)

* **Input (사용자 입력):** > "오늘 스크럼 시작할게요. 민수님은 어제 말씀하신 메인 페이지 UI 컴포넌트 다 만드셨나요? 네, 다 만들었고 오늘 테스트 예정입니다. 그리고 백엔드 쪽에서 로그인 API를 이번 주까지 완료해주셔야 프론트랑 붙여볼 수 있을 것 같아요. 영희님이 맡아주시기로 했죠? 아 네, 제가 금요일까지 끝내놓을게요. 아, 그리고 우리 다음 주 배포 일정 확인해야 하니까 다들 달력에 일정 비워두세요."
* **Output (Gemini API 결과 및 이메일 본문 내용):**
```markdown
### 📌 스크럼 회의 요약
- 프론트엔드 메인 페이지 UI 컴포넌트 개발 완료 및 오늘 테스트 진행 예정.
- 프론트-백엔드 연동을 위한 로그인 API 개발 일정 조율.

### 🎬 액션 아이템 (Action Items)
- **민수:** 메인 페이지 UI 컴포넌트 테스트 진행
- **영희:** 로그인 API 개발 완료 (~금요일까지)
- **[공통]:** 다음 주 배포 일정을 확인하고 개인 캘린더 비워두기

```

## 5. UI/UX 및 인터랙션 요구사항

### 화면 구조 (단일 레이아웃)

* **좌측 영역 (입력):** 대형 텍스트 입력창(`textarea`), `[AI 요약 및 액션 아이템 도출]` 버튼
* **우측 영역 (결과 및 전송):** - AI 결과 출력창 (마크다운 변환 뷰어 또는 텍스트 영역)
* 이메일 입력창 (`input type="text"`, placeholder: *"수신자 이메일을 쉼표(,)로 구분하여 입력하세요"*)
* `[이메일 전송]` 버튼



### 인터랙션 및 상태 관리

1. **초기 상태 (Empty):** 우측 결과 창에는 *"회의 내용을 입력하고 분석 버튼을 누르면 이곳에 결과가 표시됩니다."* 가이드 노출. `[이메일 전송]` 버튼은 비활성화 상태.
2. **로딩 상태 (Loading):** - AI 분석 중: 버튼이 `요약 중...`으로 변경되며 스피너 표시, 버튼 비활성화.
* 이메일 발송 중: 버튼이 `전송 중...`으로 변경되며 버튼 비활성화.


3. **이메일 입력값 검증 (Validation):** - `[이메일 전송]` 클릭 시, 프론트엔드에서 쉼표로 분리된 각 항목이 이메일 형식(정규식 검사 또는 `@` 포함 여부)에 맞는지 체크.
* 형식이 올바르지 않으면 `alert("올바르지 않은 이메일 형식이 포함되어 있습니다.")` 가이드 노출 후 전송 중단.


4. **성공/실패 피드백 (Feedback):**
* **성공:** 전체 발송 성공 시 `alert("지정된 팀원들에게 요약 이메일 전송을 완료했습니다.")` 알림 후 입력창 초기화.
* **실패:** 네트워크 에러나 일부 발송 실패 시, 백엔드 에러 처리를 통해 `alert("이메일 발송에 실패했습니다. 이메일 주소 및 서버 설정을 확인하세요.")` 노출 (부분 성공 예외 처리는 학습 목적상 생략하고 전체 실패로 간주).

## 6. 백엔드 및 시스템 연동 제약 사항 (ADR 참고용)

* **데이터베이스 (DB):** **사용 안 함 (No DB).** 웹앱이 켜져 있는 동안의 메모리와 프론트엔드 상태(State)로만 데이터를 유지하여 구조를 단순화합니다.
* **Gmail 인증 전제 조건 (보안):** - 발송에 사용할 구글 계정은 반드시 **2단계 인증**이 활성화되어 있어야 합니다.
* 구글 계정 설정에서 발급받은 16자리 '앱 비밀번호(App Password)'를 사용합니다.
* 이 앱 비밀번호와 Gemini API 키는 절대로 소스코드에 하드코딩하지 않으며, 백엔드의 `.env` 환경변수 파일로 분리하여 관리합니다.


* **이메일 규격:**
* **제목:** `[스크럼 요약] YYYY-MM-DD 회의 공유` (날짜는 발송 시점의 서버 날짜를 기준으로 백엔드에서 자동 생성)
* **본문:** Gemini API가 리턴한 마크다운/텍스트 데이터를 그대로 메일 본문에 매핑하여 발송 (`plain/text` 혹은 간단한 HTML 변환 포맷 활용).
