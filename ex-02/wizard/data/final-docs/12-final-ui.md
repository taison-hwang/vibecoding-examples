### 3.3 로컬 상태 및 LocalStorage 훅 (`src/hooks/useLocalStorage.ts`)
```typescript
import { useState, useEffect } from "react";

export function useLocalStorage<T>(key: string, initialValue: T) {
  const [storedValue, setStoredValue] = useState<T>(() => {
    try {
      const item = window.localStorage.getItem(key);
      return item ? JSON.parse(item) : initialValue;
    } catch (error) {
      return initialValue;
    }
  });

  useEffect(() => {
    window.localStorage.setItem(key, JSON.stringify(storedValue));
    
    /* 
      [MVP 엣지케이스 주석 및 한계 고지]
      본 훅은 본 프로젝트의 '팀원 목록 관리' 목적으로 고정된 단일 Key환경을 전제로 제작되었습니다.
      만약 런타임에 동적으로 key가 변경되는 범용 유틸리티로 확장할 경우, 
      이전 key에 해당하는 localStorage 데이터의 무덤(잔여 데이터)을 청소하는 로직(cleanup)이 추가되어야 합니다.
    */
  }, [key, storedValue]);

  return [storedValue, setStoredValue] as const;
}
```

## 4. 에러 및 예외 처리 UX 세부 명세

실제 비즈니스 로직 및 외부 연동 인프라에서 발생할 수 있는 주요 예외 상황의 출력 기준을 통일합니다.

### 4.1 Gemini 변환 실패 시
* **상태**: 원문 데이터 부족 혹은 인공지능 응답 지연/파싱 에러 감지 시.
* **UI 대응**: 결과 영역 컴포넌트 내에 연한 붉은색 알림 카드를 노출하고 `"요약 도출에 실패했습니다. 다시 시도해주세요."` 문구를 표시합니다. 
* **인터랙션**: 박스 내부에 `[재시도]` 버튼을 즉시 활성화하여 사용자가 원문을 유실하지 않고 다시 트리거할 수 있도록 배려합니다.

### 4.2 이메일 전송 결과에 따른 분기 기준 (중요 보완)
구현 연계 시 혼선 방지를 위해 **모달**과 **토스트**의 명확한 사용 기준을 아래와 같이 수립합니다.
* **전체 발송 실패 (SMTP 연동 파손 등)**:
  * **표시 방식**: **경고 모달 (Modal)** 창을 중앙에 팝업하여 경고의 엄밀함을 높입니다.
  * **문구**: `"전체 이메일 전송에 실패했습니다. 백엔드 환경변수 및 SMTP 설정을 확인하세요."`
  * **인터랙션**: 확인 클릭 시 모달이 닫히며 이메일 전송 버튼이 다시 활성화(`disabled=false`)됩니다.
* **일부 발송 실패 (특정 수신자 주소 오류 등)**:
  * **표시 방식**: **우측 상단 경고 토스트 (Toast)** 레이어로 가볍게 알립니다.
  * **문구**: `"일부 팀원에게 메일 발송을 실패했습니다. 실패 주소를 확인하세요."`

### 4.3 이메일 정규식 유효성 검증
* **검증 방식**: 기본적으로 HTML5 브라우저 자체 스펙인 `<input type="email">`을 활용해 1차 차단을 수행합니다.
* **2차 정규식 스크리닝**: 스크립트단에서 공백 검사 및 최소 규격 포맷 검증을 병행하되, 정규식(`^[^@\s]+@[^@\s]+\.[^@\s]+$`) 특성상 최상위 도메인(TLD) 누락 등의 미세 엣지케이스는 MVP 특성상 허용함을 구현 가이드 주석에 명시하고 프론트엔드 단순 패싱을 유도합니다. 유효하지 않을 시 인풋 하단에 `올바른 이메일 형식이 아닙니다` 문구를 붉은색으로 출력합니다.
