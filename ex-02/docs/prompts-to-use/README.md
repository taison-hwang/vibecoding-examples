# prompts-to-use — 순수 사용자 입력 프롬프트 모음

`docs/prompts-in-llm/` 의 대화 기록에서 **사용자가 각 LLM에 실제로 입력한 프롬프트**만 뽑아 재사용 가능한 형태로 정리한 폴더입니다.
이전 단계에서 **LLM이 생성한 산출물**이 들어가던 자리는 아래의 플레이스홀더로 표시했습니다.

```text
{{ 채워 넣어야 할 내용 설명 }}
```

> 사용법: 해당 단계를 직접 실행할 때, 앞 단계 LLM의 출력물 전문을 복사해 `{{ ... }}` 자리에 붙여넣으면 됩니다.
> `[작성지침]` 등 사용자가 직접 붙여넣는 시스템 지침은 LLM 생성물이 아니므로 원문 그대로 유지했습니다.

## 파이프라인 흐름

요구사항 정의 → ADR → UI 프로토타입을 **Gemini(생성) ↔ Claude(비판적 리뷰)** 교차 검증으로 발전시키는 구조입니다.

| 파일 | 대상 LLM | 작업 | 플레이스홀더(붙여넣을 LLM 산출물) |
|---|---|---|---|
| `01-to-gemini.md` | Gemini | 요구사항 정의서 작성 | 없음 (핵심 요구사항 + 작성지침 모두 사용자 입력) |
| `02-to-claude.md` | Claude | 요구사항 정의서 리뷰 | 01에서 Gemini가 생성한 요구사항 정의서 |
| `03-to-gemini.md` | Gemini | 리뷰 반영해 정의서 개선 | 02에서 Claude가 생성한 리뷰 |
| `05-to-gemini.md` | Gemini | ADR 작성 | 최종 요구사항 정의서 (+ 작성지침은 원문 유지) |
| `06-to-claude.md` | Claude | ADR 리뷰 | 05에서 Gemini가 생성한 ADR |
| `07-to-gemini.md` | Gemini | 리뷰 반영해 ADR 개선 | 06에서 Claude가 생성한 리뷰 |
| `09-to-gemini.md` | Gemini | UI 컨셉 프로토타입 작성 | 최종 요구사항 정의서 (+ 작성지침은 원문 유지) |
| `10-to-claude.md` | Claude | UI 프로토타입 리뷰 | 09에서 Gemini가 생성한 프로토타입 문서 |
| `11-to-gemini.md` | Gemini | 리뷰 반영해 프로토타입 개선 | 10에서 Claude가 생성한 리뷰 |

> 원본의 `04-final-urd.md` · `08-final-adr.md` · `12-final-ui.md` 는 각 단계의 **LLM 최종 출력물**이라 사용자 입력 프롬프트가 아니므로 이 폴더에는 포함하지 않았습니다.
