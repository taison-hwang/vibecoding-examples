# 프로젝트 배경 및 해결할 문제

* **발생 경위 및 현재의 불편함:** 매일 진행되는 스크럼 회의 직후, 팀원들은 회의 내용을 정리하고 액션 아이템(회의 후 개인이 수행해야 할 작업)을 도출하여 공유하는 데 추가적인 시간과 피로감을 느끼고 있습니다. 회의록 작성이 늦어지거나 누락되면 논의된 업무가 휘발되는 문제가 발생합니다.
* **이번에 풀려는 핵심 과제:** 회의 텍스트(또는 녹음 파일)를 입력하면 AI가 즉시 핵심 개요와 액션 아이템을 도출하고, 이를 한 번의 클릭으로 팀원들에게 이메일(Gmail)로 전송하여 공유 공수를 제로(0)에 가깝게 줄이는 미니 도구를 구축합니다.

# 목적 및 전체 개요

* 이 프로젝트는 바이브코딩 실습을 위한 미니 웹앱으로, 자연어 기반의 스크럼 회의 데이터를 Gemini API를 활용해 정형화된 요약본으로 변환하고 Gmail API(또는 Nodemailer 로컬 연동)를 통해 전송하는 시스템입니다.
* 복잡한 회원가입이나 팀 관리 기능 대신, 로컬에서 즉시 팀원 이메일 목록을 관리하고 결과를 확인할 수 있는 화면 2~3개 규모의 MVP를 목표로 합니다.

# 대상 사용자

* **주요 사용자:** 소규모 프로젝트 팀의 스크럼 마스터 또는 개발 팀원
* **사용 상황:** 매일 아침 10~15분간의 스크럼 회의가 끝난 직후, 회의 중 받아 적은 텍스트나 녹음된 오디오를 들고 앱에 접속해 신속하게 팀 전원에게 공유하고자 할 때 사용합니다.

# 구현 단계

## 1단계 (MVP) — 반드시 동작해야 하는 최소 기능

> **주의:** 1단계에서는 오디오 녹음/인식(STT) 등의 복잡한 기능을 배제하고, **'텍스트 입력 ➔ Gemini 요약 ➔ 이메일 발송'**의 핵심 흐름을 가짜 데이터(Mock Data)와 로컬 환경 위주로 완성합니다.

* **기능 목록:**
* 수신자 이메일 목록 등록/삭제 (브라우저 메모리 또는 LocalStorage에 저장)
* 회의 내용 텍스트 입력창 및 요약 생성 버튼
* Gemini API 연동을 통한 요약문 및 액션 아이템 UI 출력
* Gmail(Nodemailer)을 통한 결과 이메일 일괄 전송 버튼


* **화면 / 데이터 / API 개요:**
* **화면:** 단일 페이지(Single Page App) 혹은 2개 탭 구성 (1. 회의 요약 및 전송, 2. 이메일 멤버 관리)
* **데이터:** 이메일 주소 리스트 `['user1@gmail.com', 'user2@gmail.com']`
* **API:**
* `POST /api/summarize` : 회의 텍스트 수신 ➔ Gemini API 호출 ➔ 요약 및 액션 아이템 JSON 반환
* `POST /api/send-email` : 요약 텍스트 및 수신자 리스트 수신 ➔ Gmail SMTP 전송





## 2단계 — 확장 기능 (점진적 발전)

* **오디오 입력 지원 (STT):** 마이크 권한을 받아 브라우저에서 녹음하거나 Audio 파일(.mp3, .wav)을 업로드하면, Gemini API의 멀티모달 기능(Audio 직접 입력 가능)을 활용하여 텍스트 변환 없이 바로 요약 및 액션 아이템 도출.
* **이메일 템플릿 커스텀:** 전송 전 이메일 제목이나 본문 내용을 유저가 직접 UI에서 수정할 수 있는 편집 기능 추가.

# UI 요구사항

### 1. 메인 화면 (회의 요약 및 전송 탭)

* **회의 내용 입력 영역:**
* 일상 언어로 작성된 회의 텍스트를 붙여넣을 수 있는 대형 Textarea.
* *(2단계 확장용)* 마이크 녹음 버튼 및 파일 업로드 드롭존 (1단계에서는 비활성화 혹은 숨김 처리).


* **액션 버튼:** "AI 요약 및 액션 아이템 도출" 버튼 (클릭 시 로딩 스피너 표시).
* **결과 출력 영역:**
* Gemini가 리턴한 내용을 가독성 있게 렌더링 (Markdown 뷰어 권한 활용 혹은 단순 텍스트 박스).
* '개요' 섹션과 '액션 아이템 (담당자/기한 등)' 섹션으로 시각적 분리.


* **전송 버튼:** "등록된 팀원에게 이메일 전송" 버튼.
* **상태 제어:** 요약 결과가 비어있을 때는 비활성화(Disabled) 상태. 클릭 시 '전송 중...' 로딩 상태 표시 후 완료 알림(Alert).



### 2. 설정 화면 (팀원 이메일 관리 탭)

* **이메일 입력 폼:** 텍스트 인풋과 "추가" 버튼. (이메일 형식이 아닐 경우 에러 메시지 출력)
* **멤버 리스트:** 현재 등록된 이메일들이 태그(Tag) 형태로 나열되며, 각 이메일 옆에 삭제(X) 버튼 배치.
* **빈 상태(Empty State):** 등록된 이메일이 없을 때 "등록된 팀원이 없습니다. 이메일을 추가해주세요." 문구 노출.

# 기술 스택 제안

> 실습생의 개인 노트북에서 추가 비용 없이 로컬 구동이 가능한 조합입니다.

* **프론트엔드:** React + Vite + TypeScript (또는 학습자에게 익숙한 순수 HTML/JS) + Tailwind CSS (빠른 스타일링용)
* **백엔드 (API 서버):** Node.js (Express) 또는 Python (FastAPI) — *노트북 로컬 구동*
* **AI 연동:** Google AI Studio에서 발급받은 **무료 등급 Gemini API Key** (`gemini-2.5-flash` 모델 추천: 속도가 빠르고 텍스트/오디오 멀티모달 지원 처리에 적합하며 무료 한도 넉넉함)
* **이메일 전송:** `Nodemailer` (Node.js) 또는 `smtplib` (Python)
* **인증 방식:** 보안을 위해 개인 Gmail 계정의 '앱 비밀번호(App Password)'를 발급받아 로컬 `.env` 환경변수에 세팅하여 사용 (OAuth2 복잡한 설정 배제).


* **데이터 저장:** 데이터베이스(DB) 설치 없이, 브라우저의 `LocalStorage`에 이메일 리스트를 저장하여 영속성 유지.
