제공해주신 Claude의 비판적 리뷰는 실습 프로젝트의 안정성과 보안, 그리고 실제 구현 과정에서의 시행착오를 줄이기 위한 매우 날카롭고 유용한 지적들입니다.

리뷰에서 제기된 7가지 개선 사항(사실 오류 수정, API 키 보안 강화, 백엔드-이메일 기술 조합 명시, 에러 UX 정의, Gemini JSON 스키마 구체화 등)을 적극적으로 반영하여, 바이브코딩 실습 지침에 맞춘 최종 개선된 요구사항정의서(구현 지시서)를 작성했습니다.

# 프로젝트 배경 및 해결할 문제

* **발생 경위 및 현재의 불편함:** 매일 진행되는 스크럼 회의 직후, 팀원들은 회의 내용을 정리하고 액션 아이템(회의 후 개인이 수행해야 할 작업)을 도출하여 공유하는 데 추가적인 시간과 피로감을 느끼고 있습니다. 회의록 작성이 늦어지거나 누락되면 논의된 업무가 휘발되는 문제가 발생합니다.
* **이번에 풀려는 핵심 과제:** 회의 텍스트(또는 오디오 파일)를 입력하면 AI가 즉시 핵심 개요와 액션 아이템을 도출하고, 이를 한 번의 클릭으로 팀원들에게 이메일로 전송하여 공유 공수를 제로(0)에 가깝게 줄이는 미니 도구를 구축합니다.

# 목적 및 전체 개요

* 이 프로젝트는 바이브코딩 실습을 위한 미니 웹앱으로, 자연어 기반의 스크럼 회의 데이터를 Gemini API를 활용해 구조화된 JSON 데이터로 변환하고 백엔드 서버를 통해 이메일로 발송하는 시스템입니다.
* 복잡한 회원가입이나 팀 관리 기능 대신, 로컬에서 즉시 팀원 이메일 목록을 관리하고 결과를 확인할 수 있는 화면 2~3개 규모의 MVP를 목표로 합니다.

# 대상 사용자

* **주요 사용자:** 소규모 프로젝트 팀의 스크럼 마스터 또는 개발 팀원
* **사용 상황:** 매일 아침 10~15분간의 스크럼 회의가 끝난 직후, 회의 중 받아 적은 텍스트나 녹음된 오디오를 들고 앱에 접속해 신속하게 팀 전원에게 공유하고자 할 때 사용합니다.

# 구현 단계

## 1단계 (MVP) — 반드시 동작해야 하는 최소 기능

> **주의:** 1단계에서는 오디오 녹음/인식(STT) 등의 복잡한 기능을 배제하고, **'텍스트 입력 ➔ 백엔드経유 Gemini 요약 ➔ 이메일 발송'**의 핵심 흐름을 가짜 데이터(Mock Data)와 로컬 환경 위주로 완성합니다.

* **기능 목록:**
* 수신자 이메일 목록 등록/삭제 (브라우저 LocalStorage에 저장)
* 회의 내용 텍스트 입력창 및 요약 생성 버튼
* 백엔드 API를 통한 Gemini API 호출 및 정형화된 요약문 데이터 수신
* 이메일 발송 버튼 및 실패/성공 예외 처리 기능


* **화면 / 데이터 / API 개요:**
* **화면:** 단일 페이지(Single Page App) 혹은 2개 탭 구성 (1. 회의 요약 및 전송, 2. 이메일 멤버 관리)
* **데이터:** 이메일 주소 리스트 `['user1@gmail.com', 'user2@gmail.com']`
* **백엔드 통신 계약 (API 엔드포인트 명세):**
* `POST /api/summarize` : 회의 텍스트 수신 ➔ 백엔드에서 Gemini API 호출 ➔ 아래 [Gemini 응답 스키마]에 맞춘 JSON 반환
* `POST /api/send-email` : 요약 텍스트 및 수신자 리스트 수신 ➔ 이메일 서버를 통해 일괄 발송





### [Gemini / API 응답 스키마 계약]

백엔드와 프론트엔드가 주고받는 데이터 구조를 아래와 같이 통일하여 파싱 에러를 방지합니다. Gemini 프롬프트 작성 시 `responseSchema`를 설정하거나 JSON 출력을 강제해야 합니다.

```json
{
  "summary": "오늘 스크럼 회의의 핵심 내용을 요약한 텍스트입니다.",
  "action_items": [
    { "owner": "홍길동", "task": "백엔드 API 명세서 작성 및 공유", "due": "2026-07-01" },
    { "owner": "이영희", "task": "프론트엔드 설정 페이지 UI 마크업", "due": "2026-07-02" }
  ]
}

```

## 2단계 — 확장 기능

* **오디오 파일 처리 (STT 대체):** 브라우저에서 직접 실시간 스트리밍 녹음을 전송하는 방식은 API 제약이 따르므로, 브라우저에서 녹음 완료 후 **오디오 파일(.mp3, .wav 등) 형태로 백엔드에 업로드**합니다. 백엔드에서는 `Gemini File API`를 경유해 오디오 파일을 Gemini에 직접 전달하여 텍스트 변환 없이 요약 및 액션 아이템을 도출합니다.
* **이메일 템플릿 커스텀:** 전송 전 이메일 제목이나 본문 내용을 유저가 직접 UI에서 수정할 수 있는 편집 기능 추가.

# UI 요구사항 및 예외 처리(UX)

### 1. 메인 화면 (회의 요약 및 전송 탭)

* **회의 내용 입력 영역:** 일상 언어로 작성된 회의 텍스트를 붙여넣을 수 있는 대형 Textarea.
* **액션 버튼:** "AI 요약 및 액션 아이템 도출" 버튼.
* **인터랙션:** 클릭 시 버튼 내부에 로딩 스피너가 표시되며, 완료될 때까지 버튼은 비활성화(Disabled) 처리됩니다.


* **결과 출력 영역:** Gemini가 리턴한 JSON 데이터를 바탕으로 '개요' 섹션과 '액션 아이템 테이블(담당자, 업무, 기한)'로 시각적으로 정돈하여 렌더링합니다.
* **전송 버튼:** "등록된 팀원에게 이메일 전송" 버튼.
* **상태 제어:** 요약 결과가 없거나 수신자 이메일 목록이 비어있을 때는 버튼이 비활성화됩니다.



### 2. 에러 및 예외 처리 UX (중요 보완)

* **Gemini 변환 실패 시:** AI 응답 지연이나 파싱 에러 발생 시, 결과 영역에 에러 메시지("요약 도출에 실패했습니다. 다시 시도해주세요.")와 함께 [재시도] 버튼을 노출합니다.
* **이메일 전송 실패 시:** SMTP 인증 실패, 네트워크 오류, 잘못된 수신자 주소 등으로 인해 전송이 실패한 경우, 단순 성공 알림 대신 "일부 또는 전체 이메일 전송에 실패했습니다. 백엔드 설정을 확인하세요."라는 경고 모달/토스트 메시지를 띄우고 전송 버튼을 다시 활성화합니다.

### 3. 설정 화면 (팀원 이메일 관리 탭)

* **이메일 입력 폼:** 텍스트 인풋과 "추가" 버튼. 정규식을 통해 이메일 형식이 아닐 경우 "올바른 이메일 형식이 아닙니다"라는 에러 문구를 인풋 하단에 붉은색으로 출력합니다.
* **멤버 리스트 / 빈 상태:** 등록된 이메일이 태그 형태로 나열되며 [X] 버튼으로 삭제 가능합니다. 등록된 데이터가 없으면 "등록된 팀원이 없습니다."라는 안내 문구를 노출합니다.

# 기술 스택 및 보안 지침

### 1. ⚠️ 크리티컬 보안 지침 (API 키 노출 방지)

* **Gemini API 키는 절대로 프론트엔드(React/Vite) 코드나 환경변수(`VITE_`)에 포함시켜 브라우저에 노출해서는 안 됩니다.**
* API 키는 반드시 백엔드 서버의 `.env` 파일에만 안전하게 보관하고, 프론트엔드는 백엔드 엔드포인트(`/api/summarize`)를 통해서만 AI 기능을 호출하도록 아키텍처를 강제합니다.

### 2. 백엔드 및 이메일 기술 스택 조합

실습생의 혼선을 방지하기 위해 백엔드 언어에 따른 이메일 라이브러리 조합을 명확히 분리합니다. 실습 환경에 맞춰 한 가지 조합을 선택하여 진행합니다.

| 백엔드 선택 (택 1) | 이메일 연동 라이브러리 | 환경 변수 관리 |
| --- | --- | --- |
| **Node.js** (Express) | `Nodemailer` | `dotenv` |
| **Python** (FastAPI) | `smtplib` + `email.mime` | `python-dotenv` |

### 3. 이메일 발송 환경 전제 조건 및 대안

Gmail SMTP를 사용하여 메일을 발송할 때의 제약사항을 인지하고 환경을 준비합니다.

* **Gmail SMTP 사용 시:** 발송용 Google 계정에 **'2단계 인증'이 반드시 활성화**되어 있어야 하며, Google 계정 설정에서 '앱 비밀번호(App Password)'를 생성하여 백엔드 환경변수에 세팅해야 합니다.
* **인증 방식 대안:** 실습 계정의 보안 설정 변경이 어렵거나 Google 정책 변화로 앱 비밀번호 발급이 원활하지 않을 경우, 개발자용 무료 SMTP 테스트 서비스인 **Mailtrap**이나 **Resend**(무료 등급)를 대안으로 연동하여 실습을 진행합니다.

### 4. 프론트엔드 및 데이터 저장

* **프론트엔드:** React + Vite + TypeScript + Tailwind CSS
* **AI 모델:** `gemini-2.5-flash` (텍스트 및 오디오 처리에 최적화되어 있으며, 실습 규모에서는 대개 무료 한도 내에서 안정적으로 동작함. 상세 한도는 Google AI Studio 정책 참조)
* **데이터 저장:** 복잡한 DB 구축 없이, 브라우저의 `LocalStorage`를 활용해 팀원 이메일 목록의 영속성을 유지합니다.
